Exploring Defects: A Comparative Analysis in Open-Source Projects Using Siesta Specification


This study aims to introduce an approach called SIESTA (Simple Imperative Specification Language). The SIESTA proposal aims to provide developers with a simple way to specify the semantics of the programming language they already master, aligning it directly and understandably with the semantics of the specification language. This provides a more minimalist integration between specification descriptions and the programming language used, simplifying the development process and promoting a clearer and more accessible understanding of code semantics.

2. Choose of Project

The Greplin Zookeeper Utilities project was used as an example, but you can choose any Maven-based project. The choice of the zookeeper-utils project was guided by the findings presented in the article 'How Good Are the Specs? A Study of the Bug-Finding Effectiveness of Existing Java API Specifications.' 3.Runtime Verification of Specifications

Let's configure and run the SIESTA and JAVAMOP approaches, which aim to verify runtime violations in the project under analysis. For this purpose, we provide two Maven extension libraries for integration, enabling Maven to perform runtime analyses during test execution.

First, we'll set up and then run SIESTA. To begin, we'll transfer the siesta-mop-extension.jar file to the folder: '../..Maven.../lib/ext' 

 cp /home/mopuser/mop-extension/target/mop-extension-1.0-SNAPSHOT.jar /home/mopuser/apache-maven-3.9.6/lib/ext/

After copying the library to the Maven environment, we need to update the pom.xml file with the dependencies required to execute the mop-extension.jar. This will ensure that Maven can find and utilize the newly added library during the project's build and execution process. 

wget https://gist.githubusercontent.com/leosoftlima/e23b740f58ead19de34f7fcde0c8ed79/raw/78e72c6fb1ab0fe917975e118b8f34980f354f5c/pom-mop.diff

Let's run the tests and automatically check for violations. If any are found, they will be saved in the log file. 

mvn test -fn -e > vio-javamop.log

To execute the SIESTA approach, it's necessary to modify the content of the file "/home/mopuser/workspace/loadLibrary.csv" and replace the word "javamop" with "siestamop"To execute the SIESTA approach, it's necessary to modify the content of the file "/home/mopuser/workspace/loadLibrary.csv" and replace the word "javamop" with "siestamop"

Afterwards, execute the command:

 mvn test -fn -e > vio-siesta.log
